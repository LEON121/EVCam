# EVCam 系统功能与接口梳理

## 1. 项目基本信息

| 项 | 说明 |
| --- | --- |
| 项目名称 | EVCam |
| 包名 | `com.kooo.evcam` |
| 技术栈 | Java + Android + Camera2 + OkHttp + WorkManager |
| 最低版本 | Android 9.0 / API 28 |
| 目标版本 | API 36 |
| 主入口 | `MainActivity` |
| 核心目标 | 多摄像头行车记录、远程查看、补盲显示、车机场景保活 |

当前仓库为单模块 Android 工程，核心代码位于 `app/src/main/java/com/kooo/evcam`。系统围绕“多摄像头预览/录制/拍照”展开，并扩展了远程控制、心跳推图、补盲联动、文件分享、保活和升级能力。

## 2. 页面与功能入口

- 主界面：`MainActivity`，负责预览、录制、拍照、远程指令落地、状态展示。
- 视频回放：`PlaybackFragmentNew`，支持按日期分组、多路同步预览、倍速、批量删除/分享。
- 图片回放：`PhotoPlaybackFragmentNew`，支持按日期分组、多路/单路查看、批量删除/分享。
- 钉钉远程：`RemoteViewFragment`。
- Telegram 远程：`TelegramFragment`。
- 飞书远程：`FeishuFragment`。
- 心跳推图：`heartbeat/HeartbeatFragment`。
- 软件设置：`SettingsFragment`。
- 权限设置：`PermissionSettingsFragment`。
- 分辨率/码率/帧率与图像调节：`ResolutionSettingsFragment`。
- 自定义车型与布局：`CustomCameraConfigFragment`。
- 补盲设置：`BlindSpotSettingsFragment`、`BlindSpotCorrectionFragment`、`BlindSpotLabFragment`、`SecondaryBlindSpotAdjustFragment`。
- 辅助页面：`PhotoViewerActivity`、`VideoPlayerActivity`、`LogcatViewerActivity`。

## 3. 核心功能清单

### 3.1 预览、录制、拍照

- 最多支持 4 路摄像头同时预览、录制、拍照。
- 支持车型预设：银河 E5、E5 多按钮、L6/L7、L7 多按钮、26 款星舰 7、手机、自定义车型、多视角布局。
- 支持双录制管线：
  - `VideoRecorder`：MediaRecorder。
  - `CodecVideoRecorder` + `EglSurfaceEncoder`：OpenGL + MediaCodec。
- 支持录制分段：1/3/5 分钟。
- 支持录制摄像头选择、画面显隐分离控制。
- 支持时间角标、水印、录制状态显示、录制悬浮按钮。
- 支持启动自动录制、息屏录制、后台录制恢复、远程录制不分段覆盖。
- 支持图像调节：曝光补偿、白平衡、色调映射、边缘增强、降噪、特效。
- 支持预览矫正、鱼眼矫正、补盲画面矩阵矫正。

### 3.2 回放与分享

- 视频与图片均按“日期 -> 时间戳组”聚合。
- 支持四宫格预览与单路放大切换。
- 视频支持播放/暂停、拖动进度、倍速播放。
- 支持多选、全选、批量删除。
- 支持两种分享方式：
  - 系统分享：基于 `FileProvider`。
  - 扫码互传：本地 HTTP 文件服务 + 二维码。

### 3.3 远程控制

- 已实现三类远程平台：
  - 钉钉：Stream SDK。
  - Telegram：Bot API 轮询。
  - 飞书：Bot + WebSocket。
- 远程能力统一收敛到 `remote/RemoteCommandDispatcher`。
- 支持功能：
  - 查询状态。
  - 唤醒到前台。
  - 切到后台。
  - 启动持续录制。
  - 停止持续录制。
  - 远程录制指定秒数视频并上传。
  - 远程拍照并上传。
  - 退出应用、二次确认退出。
- 平台差异：
  - Telegram 支持 `allowedChatIds` 白名单，视频上传受 50MB 限制。
  - 飞书支持 `allowedUserIds` 白名单，视频上传受 30MB 限制。
  - 钉钉配置 `ClientId/ClientSecret`，支持自动重连。

### 3.4 心跳推图

- 周期性抓取当前多路预览画面，拼接并压缩后推送到指定服务器。
- 支持亮屏推图、息屏推图、自动启动。
- 支持 30/60/120/300 秒周期。
- 图片大小支持 100KB、500KB、1MB、原图。
- 自动生成固定 `vehicleId`，并附带状态 JSON、存储状态、录制状态等信息。

### 3.5 补盲、联动与超视

- 转向灯联动补盲，支持三种触发源：
  - `logcat`
  - `vhal_grpc`
  - `car_signal_manager`
- 支持车门联动、长视模式、超视模式双悬浮窗。
- 支持主屏悬浮补盲、副屏显示、主屏复用、独立补盲窗定位。
- 支持全景影像避让，避免与指定 Activity/包名前台冲突。
- 支持状态栏动效、颜色、透明度、圆角、长按拖动、模拟转向灯悬浮按钮。
- 支持自定义按键唤醒预览。

### 3.6 存储、传输与升级

- 支持内部存储与 U 盘存储。
- 支持“中转写入”：先写 `cache/temp_video`，再异步转存到最终目录，降低慢速 U 盘对录制的影响。
- 支持按容量上限自动清理视频和图片，内部存储低于 3GB 时会触发强制清理。
- 支持版本检查与 APK 下载，更新服务器约定：
  - `version.txt`
  - `EVCam-v{version}-release.apk`

### 3.7 保活、权限与诊断

- 保活链路包括：
  - `BootReceiver`
  - `CameraForegroundService`
  - `KeepAliveProvider`
  - `KeepAliveReceiver`
  - `KeepAliveWorker`
  - `KeepAliveAccessibilityService`
  - WakeLock + 电池优化白名单
- 权限页支持相机、麦克风、存储、通知、悬浮窗、无障碍、Usage Stats、所有文件访问、忽略电池优化、一键 ADB 授权。
- 支持日志实时查看、日志保存、日志上传到 Gotify 服务。
- 支持 E245 系统白名单一键配置与恢复。

## 4. 接口清单

### 4.1 远程命令接口

| 平台 | 命令 |
| --- | --- |
| 钉钉 | `状态`、`前台`、`后台`、`启动录制`、`结束录制`、`录制`、`录制30`、`拍照`、`退出`、`确认退出`、`帮助` |
| Telegram | 中文命令同上，另支持 `/status`、`/foreground`、`/background`、`/start_rec`、`/stop_rec`、`/record 30`、`/photo`、`/exit`、`/confirm_exit`、`/help` |
| 飞书 | 中文命令同上，另支持 `status`、`foreground`、`background`、`start`、`stop`、`record`、`photo`、`exit`、`help` |

### 4.2 Activity 唤醒参数

`WakeUpHelper` 通过启动 `MainActivity` 传递远程动作，关键参数如下：

- `remote_action`：`record`、`photo`、`start_recording`、`stop_recording`、`foreground`
- `remote_wake_up`：是否为后台唤醒
- `remote_source`：`telegram`、`feishu`
- 钉钉参数：`remote_conversation_id`、`remote_conversation_type`、`remote_user_id`
- Telegram 参数：`telegram_chat_id`
- 飞书参数：`feishu_chat_id`、`feishu_message_id`
- 时长参数：`remote_duration`

### 4.3 Service Action / 广播接口

#### `CameraRecordingService`

- `com.kooo.evcam.action.START_RECORDING`
- `com.kooo.evcam.action.STOP_RECORDING`
- `com.kooo.evcam.action.START_BLIND_SPOT`
- `com.kooo.evcam.action.STOP_BLIND_SPOT`

#### `BlindSpotOverlayService`

- `com.kooo.evcam.action.SHOW_BLIND_SPOT_OVERLAY`
- `com.kooo.evcam.action.HIDE_BLIND_SPOT_OVERLAY`
- `com.kooo.evcam.action.UPDATE_BLIND_SPOT_LAYOUT`

附带参数：

- `side`：`left` / `right`
- `x`、`y`、`width`、`height`

#### 其他广播

- `com.kooo.evcam.SUPERVISION_MODE_CHANGED`
- `com.kooo.evcam.ACTION_MOVE_TO_BACKGROUND`
- `com.kooo.evcam.ACTION_KEEP_ALIVE`

### 4.4 本地 HTTP 文件传输接口

`transfer/FileTransferServer` 提供本地文件浏览与下载接口：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/` | 文件列表网页 |
| `GET` | `/index.html` | 文件列表网页 |
| `GET` | `/api/files` | 预设待传文件 JSON |
| `GET` | `/api/browse?path=/xxx` | 浏览本地目录 |
| `GET` | `/download/{fileId}` | 下载文件 |
| `OPTIONS` | `*` | CORS 预检 |

### 4.5 心跳推图 HTTP 接口

`heartbeat/HeartbeatApiClient` 以 `POST` 方式向配置的 `serverUrl` 发送请求。

请求头：

- `X-Vehicle-Id`
- `X-Timestamp`
- `X-Nonce`
- `X-Signature`

请求体核心字段：

```json
{
  "vehicleId": "EV-XXXXXXX",
  "timestamp": 0,
  "nonce": "uuid",
  "signature": "hmac_sha256",
  "imageBase64": "...",
  "imageWidth": 0,
  "imageHeight": 0,
  "imageSizeBytes": 0,
  "cameraCount": 0,
  "status": {}
}
```

签名规则：`HMAC-SHA256(vehicleId + timestamp + nonce, secretKey)`。

### 4.6 日志上传接口

`AppLog.uploadLogsToServer()` 将日志以 JSON 形式发送到内置 Gotify 服务，内容包括：

- 设备昵称
- 设备型号 / Android 版本 / App 版本
- 问题描述
- 当前或上次运行日志全文

## 5. 配置与持久化

主要使用 `SharedPreferences` 保存配置：

- `app_config`：主配置、车型、存储、补盲、悬浮窗、升级地址等
- `recording_config`：新录制架构参数
- `blind_spot_config`：补盲窗口与裁剪参数
- `dingtalk_config`
- `telegram_config`
- `feishu_config`
- `heartbeat_config`

## 6. 关键存储路径

- 视频目录：`/sdcard/DCIM/EVCam_Video/`
- 图片目录：`/sdcard/DCIM/EVCam_Photo/`
- 日志目录：`/sdcard/Download/EVCam_Log/`
- 临时视频目录：`{cache}/temp_video/`
- U 盘模式下：写入 U 盘根目录下的 `DCIM/EVCam_Video`、`DCIM/EVCam_Photo`

## 7. 结论

当前 EVCam 已不是单一“行车记录仪”应用，而是一个围绕车机多摄像头运行的综合系统，已形成以下能力闭环：

- 本地预览、录制、拍照、回放、分享。
- 远程控制、远程上传、状态回报。
- 补盲联动、副屏/悬浮窗显示、超视与长视扩展。
- 心跳推图、升级、日志、权限、保活、车机适配。
