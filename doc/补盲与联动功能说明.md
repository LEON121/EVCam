# EVCam 补盲与联动功能说明

## 1. 功能概述

补盲系统是 EVCam 中与车机使用场景强相关的扩展模块，目标是在转向、开门、超视、长视等场景下，自动把侧向画面以悬浮窗或副屏方式显示出来，并允许用户持续调节裁剪、鱼眼、矫正、位置和动效。

当前补盲体系同时覆盖：

- 转向灯联动补盲
- 车门联动补盲
- 主屏悬浮窗
- 副屏补盲显示
- 预览补盲窗口
- 超视模式双窗
- 长视模式双窗
- 模拟转向灯悬浮按钮
- 自定义按键唤醒预览
- 全景影像避让
- 补盲画面裁剪与矫正

## 2. 核心模块

| 类/模块 | 作用 |
| --- | --- |
| `BlindSpotService` | 补盲总控服务，负责窗口、信号监听、联动逻辑 |
| `BlindSpotSettingsFragment` | 补盲主设置页 |
| `BlindSpotCorrectionFragment` | 补盲矩阵矫正页 |
| `BlindSpotLabFragment` | 主屏补盲实验页 |
| `SecondaryBlindSpotAdjustFragment` | 副屏位置调整 |
| `BlindSpotFloatingWindowView` | 侧向补盲悬浮窗 |
| `MainFloatingWindowView` | 主屏常驻悬浮预览 |
| `BlindSpotStatusBarView` | 补盲状态栏动效 |
| `BlindSpotConfig` | 新补盲配置对象 |
| `AppConfig` | 老补盲、联动、超视、长视、避让等全局配置 |

## 3. 入口与控制面板

### 3.1 页面入口

主导航入口：

- `nav_secondary_display` -> `BlindSpotSettingsFragment`
- `nav_supervision_mode` -> 直接切换超视模式

补盲设置页可继续进入：

- `BlindSpotLabFragment`
- `BlindSpotCorrectionFragment`
- `SecondaryBlindSpotAdjustFragment`

### 3.2 全局开关

补盲体系以 `appConfig.isBlindSpotGlobalEnabled()` 为总开关。

关闭总开关时：

- 子功能区域隐藏
- `BlindSpotService` 被停止

开启总开关时：

- 会重新调用 `BlindSpotService.update(context)` 刷新服务状态

## 4. 触发源与联动模式

### 4.1 转向灯触发源

当前支持三种来源：

| 模式 | 常量 | 场景 |
| --- | --- | --- |
| Logcat | `TRIGGER_MODE_LOGCAT` | 通过系统日志关键字触发 |
| 车辆 API | `TRIGGER_MODE_VHAL_GRPC` | 银河 E5 / 26 款星舰 7 |
| CarSignalManager | `TRIGGER_MODE_CAR_SIGNAL_MANAGER` | 银河 L6/L7 |

### 4.2 预设

`BlindSpotSettingsFragment` 中可选择：

- L6/L7 预设
- 博越 L 预设
- 星舰 7 日志预设
- 自定义关键字
- 车辆 API 模式

如果是 `logcat` 模式，还可以填写：

- 左转触发关键字
- 右转触发关键字

### 4.3 车门联动

车门联动依赖车型/触发源：

- 车辆 API 模式支持
- CarSignalManager 模式支持

不支持的触发模式下，UI 会自动隐藏或关闭车门联动开关。

### 4.4 长视模式

长视模式是双窗联动显示的一种扩展模式。

特点：

- 开启后由转向灯触发双画面
- 可以进入专用调整模式
- 实际上复用了超视窗口的调节能力

### 4.5 超视模式

超视模式通过以下两种方式启动：

- 导航菜单直接切换
- 向 `BlindSpotService` 发送广播 `com.kooo.evcam.SUPERVISION_MODE_CHANGED`

服务内部也支持以下显式 action：

- `START_SUPERVISION_MODE`
- `STOP_SUPERVISION_MODE`

超视模式激活后：

- 左右两个悬浮补盲窗常驻显示
- `CameraForegroundService` 会被保持运行

## 5. BlindSpotService 的动作接口

`BlindSpotService.onStartCommand()` 当前识别以下控制指令。

### 5.1 通过 `intent.getStringExtra("action")`

| action | 说明 |
| --- | --- |
| `setup_blind_spot_window` | 打开补盲位置设置窗口 |
| `preview_blind_spot` | 打开指定摄像头的预览补盲窗口 |
| `stop_preview_blind_spot` | 关闭预览补盲窗口 |
| `enter_secondary_display_adjust` | 进入副屏调整模式 |
| `exit_secondary_display_adjust` | 退出副屏调整模式 |

相关参数：

- `camera_pos`：`front` / `back` / `left` / `right`

### 5.2 其他额外参数

- `mock_turn_signal`：触发模拟转向灯逻辑

### 5.3 直接读取 `Intent.getAction()`

- `START_SUPERVISION_MODE`
- `STOP_SUPERVISION_MODE`

## 6. 窗口类型

### 6.1 主屏常驻悬浮窗

由 `MainFloatingWindowView` 管理。

功能：

- 常驻显示指定摄像头画面
- 支持位置、大小、圆角、长按拖动
- 可被全景影像避让逻辑临时隐藏

### 6.2 专用补盲悬浮窗

由 `BlindSpotFloatingWindowView` 管理。

场景：

- 左转/右转时临时弹出
- 超视模式双窗
- 长视模式双窗
- 预览调试窗

### 6.3 副屏补盲窗口

`BlindSpotService` 可通过 `DisplayManager` 和 `WindowManager` 在副屏上显示指定摄像头画面。

可配置项包括：

- 显示开关
- Camera 位置
- Display ID
- X/Y
- 宽高
- 旋转角度
- 透明度
- 边框显示

### 6.4 模拟转向灯悬浮按钮

用于无真实车机信号时手动触发补盲联动。

支持：

- 显示/隐藏
- 拖动保存位置
- 点击模拟左/右信号

## 7. 避让逻辑

### 7.1 全景影像避让

配置项：

- `avm_avoidance_enabled`
- `avm_avoidance_activity`

工作方式：

- 定时检测指定 Activity 是否在前台
- 如果目标前台存在，则隐藏主屏补盲窗
- 如果目标退到后台，再恢复补盲窗

特殊值：

- `all`：始终避让，即主屏补盲窗口常驻隐藏

### 7.2 EVCam 自身前后台避让

`BlindSpotService` 同时会结合 `MainActivity.onResume/onPause` 更新的前后台状态决定是否显示主屏补盲窗，避免和自身前台页面冲突。

## 8. 配置项分层

### 8.1 AppConfig 中的主要补盲配置

这一层偏“业务行为控制”：

- 全局开关
- 转向灯联动开关
- 车门联动开关
- 超视模式开关
- 长视模式开关
- 副屏开关
- 主屏悬浮窗开关
- 转向灯超时
- 主屏/副屏位置与大小
- 模拟转向灯悬浮按钮
- 自定义按键唤醒
- 状态栏动效样式/颜色/透明度
- 主屏窗口圆角、比例锁定、长按拖动
- 全景影像避让

### 8.2 BlindSpotConfig 中的主要配置

这一层偏“画面参数”：

- 左右裁剪区域
- 左右鱼眼参数
- 窗口宽高
- 窗口 X/Y
- 左右叠加层旋转
- 是否启用圆角

## 9. 矫正与调参能力

### 9.1 补盲矩阵矫正

`BlindSpotCorrectionFragment` 支持按摄像头分别调整：

- `scaleX`
- `scaleY`
- `translateX`
- `translateY`
- `rotation`
- 水平镜像
- 垂直镜像

进入页面后会调用：

- `preview_blind_spot`

离开页面时会调用：

- `stop_preview_blind_spot`

### 9.2 预览矫正与鱼眼矫正

这两项入口在 `SettingsFragment` 中：

- 预览画面矫正
- 鱼眼矫正

特点：

- 可打开悬浮调节窗口
- 调整后即时刷新预览
- 支持重置所有参数

### 9.3 BlindSpotLab

实验页偏向主屏悬浮补盲能力验证，支持：

- 主屏补盲开关
- 主屏摄像头选择
- 是否复用主屏浮窗
- 进入补盲位置搭建窗口

## 10. 自定义按键唤醒

由 `VhalSignalObserver` + `AppConfig` 配合实现。

关键配置：

- `custom_key_wakeup_enabled`
- `custom_key_speed_threshold`
- `custom_key_speed_prop_id`
- `custom_key_button_prop_id`

行为：

- 当车速达到阈值且接收到指定按钮属性变化时
- 在预览与后台之间切换

这部分逻辑同样由 `BlindSpotService` 托管。

## 11. 服务状态自愈

`BlindSpotService` 的一个重要职责不是只“显示窗口”，而是保证信号监听不中断。

其自愈逻辑包括：

- `ensureSignalObserversAlive()`
- 根据当前触发模式重新初始化观察者
- 在服务更新时重新加载配置

当前可恢复的观察者：

- `LogcatSignalObserver`
- `VhalSignalObserver`
- `CarSignalManagerObserver`
- `DoorSignalObserver`

## 12. 与前台服务的关系

补盲能力依赖摄像头持续工作，因此会和 `CameraForegroundService` 协同。

典型场景：

- 超视模式激活时保持前台服务
- 窗口仍在显示时尽量不要释放摄像头
- 从后台回到前台时尝试恢复窗口摄像头预览

`BlindSpotService.hasActiveCameraWindows()` 用于判断当前是否存在仍占用摄像头的补盲窗口。

## 13. 实现注意事项

### 13.1 权限前置

以下能力必须先拿到悬浮窗权限：

- 转向灯联动补盲
- 主屏悬浮补盲
- 超视模式
- 长视模式
- 模拟转向灯按钮
- 预览补盲调试

### 13.2 车型与触发源强绑定

当前补盲联动与车型高度绑定，新增车型时应重点检查：

- 转向灯来源
- 车门信号来源
- 左右摄像头位置映射
- 是否需要鱼眼/旋转/裁剪预设

### 13.3 避免 UI 与服务配置分裂

补盲相关配置很多分散在 `AppConfig` 与 `BlindSpotConfig`，后续如继续扩展，建议按“行为配置 / 画面配置”继续保持分层，不要再把所有参数都堆回 `AppConfig`。
